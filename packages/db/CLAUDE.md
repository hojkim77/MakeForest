# DB — 스키마 & 데이터 모델

## 핵심 엔티티

**User** — 유저 프로필, 소속 동네, 가입일, 스트릭

**Neighborhood** — 읍/면/동 단위, 소속 시/구 참조

**DailySession** — 유저별 날짜별 누적 집중 시간, 물주기 횟수 (최대 3)

**TimerEvent** — 타이머 시작/종료 이벤트 로그 (서버 기준 시각 기록)
- 재접속 복구 및 자정 분리 계산에 사용

**Creature** — 날짜별 동네별 오늘의 생명체
- 종류, 현재 진화 단계, 소속 동네, 날짜
- 진화 임계값은 별도 Config 테이블에서 참조

**Fossil** (박제된 생명체) — 숲에 영구 기록
- 생명체 종류, 최종 단계, 소속 동네, 날짜
- 시/구 숲 내 좌표 (x, y)

**WateringLog** — 유저별 물주기 이력 (도감/통계용)

## 스키마 설계 원칙

- 시간 컬럼은 전부 UTC 저장, 비즈니스 로직에서 KST 변환
- 진화 임계값은 하드코딩 금지 → Config 테이블 또는 별도 관리
- Fossil은 Creature와 분리된 테이블 (영구 불변 레코드)
- TimerEvent는 집계용이 아닌 로그 테이블 — 집계는 DailySession에 반영
